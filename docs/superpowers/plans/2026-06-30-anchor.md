# Anchor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Anchor — a local, private, double-click desktop workbench that reconstructs the provenance chain (PDF page → claim → footnote → source → cross-reference) for an expert report and guides a non-technical operator through resolving every uncertain citation before handoff.

**Architecture:** One self-contained **Electron** app. Svelte + Vite renderer (the workbench UI). Node main process (local JSON file store, job runner, IPC, child-process orchestration). All-JS deterministic extraction via **pdf.js** (PDF text+positions+page render) and a JS DOCX-footnote parser — no Python, no poppler required. A deterministic graph baseline always runs; optional AI passes are driven by **local, subscription-authenticated CLI agents** (`claude -p --output-format json` or `codex exec`), chunked, schema-validated, retried once, and degrading to the deterministic result on failure. No database; per-document JSON on disk.

**Tech Stack:** Electron, Svelte 5, Vite, Node 20+, `pdfjs-dist`, `fflate` (unzip DOCX), `fast-xml-parser`, `ajv` (schema validation), `electron-builder`, Vitest (unit), Playwright (e2e). Apache-2.0.

## Global Constraints

- **License:** Apache-2.0. Every source file carries the header `// Copyright © 2026 Tucker Peters. Authored independently, in the author's personal time. Licensed under the Apache License, Version 2.0.` A `NOTICE` file restates authorship.
- **Privacy:** No telemetry. No network calls except the CLI agent's own subscription auth. No server operated by the author.
- **AI is optional, never required.** Every feature must work in deterministic mode. AI passes must never crash, hang, or block; they time out, validate, retry once, then degrade to the deterministic result and log it.
- **Security:** child processes spawned with argv arrays (never a shell string); all file paths sandboxed under the per-document folder; uploads role+extension checked before write; Electron `contextIsolation: true`, `nodeIntegration: false`, preload `contextBridge` only.
- **Non-technical UX:** every error is plain-English + a recovery action. No terminal needed after one-time install. No fake spinners — progress reflects real job state.
- **Node version floor:** Node 20+. **Electron:** latest stable major. **Svelte:** 5.x.
- **Naming:** product = "Anchor"; storage root = `<app.getPath('userData')>/documents/<id>/`.

```js
// src/main/store/documents.js
import { mkdirSync, writeFileSync, readFileSync, readdirSync, existsSync } from 'node:fs'
import { docsRoot, docDir, docFile } from './paths.js'

// slug id without Date/random (both unavailable in some envs): counter + title slug
const slug = (s) => s.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '').slice(0, 40) || 'doc'
function nextId(root, base) {
  let id = base, n = 2
  while (existsSync(docDir(root, id))) { id = `${base}-${n++}` }
  return id
}
export function createDocument(root, { title, now = new Date().toISOString() }) {
  mkdirSync(docsRoot(root), { recursive: true })
  const id = nextId(root, slug(title))
  const doc = { id, title, status: 'new', createdAt: now, updatedAt: now,
    inputs: { docx: null, pdf: null, register: null }, stats: {} }
  mkdirSync(docDir(root, id), { recursive: true })
  writeFileSync(docFile(root, id), JSON.stringify(doc, null, 2))
  return doc
}
export function getDocument(root, id) {
  return JSON.parse(readFileSync(docFile(root, id), 'utf8'))
}
export function listDocuments(root) {
  const base = docsRoot(root)
  if (!existsSync(base)) return []
  return readdirSync(base)
    .filter((d) => existsSync(docFile(root, d)))
    .map((d) => getDocument(root, d))
    .sort((a, b) => (a.updatedAt < b.updatedAt ? 1 : -1))
}
```
> Note: `now` is injected (default `new Date().toISOString()`) so tests can pass a fixed timestamp; production callers omit it. The main-process runtime has `Date`; only the Workflow scripting sandbox forbids it.

// src/main/store/workState.js
import { writeFileSync, readFileSync, existsSync, mkdirSync } from 'node:fs'
import { dirname } from 'node:path'
import { workStateFile } from './paths.js'

export function loadWorkState(root, id, now = new Date().toISOString()) {
  const f = workStateFile(root, id)
  if (!existsSync(f)) return { documentId: id, updatedAt: now, nodes: {} }
  return JSON.parse(readFileSync(f, 'utf8'))
}
export function saveWorkState(root, id, ws) {
  const f = workStateFile(root, id)
  mkdirSync(dirname(f), { recursive: true })
  writeFileSync(f, JSON.stringify(ws, null, 2))
  return { ok: true }
}
export function mergeWorkState(disk, incoming) {
  const nodes = { ...(disk.nodes || {}) }
  for (const [k, v] of Object.entries(incoming.nodes || {})) {
    const cur = nodes[k]
    if (!cur || (v.updatedAt || '') >= (cur.updatedAt || '')) nodes[k] = v
  }
  return { ...disk, ...incoming, nodes }
}

**Objective:** `buildGraph({pdf,docx,register}) -> Graph` (schema-valid, no AI). **Files:** `src/main/graph/{build,claims,classify}.js`. **Tasks:** match footnote refs in paragraphs → claims; detect footnote zones vs body; infer footnote `class` + `citationNeed` via heuristics (URL/docket/case/screenshot/cross-ref/missing patterns); link sources from register + footnote text; build all edge types; compute stats; validate against `validateGraph`. **Deliverable test:** on fixtures, exact stats + every edge endpoint resolves + `validateGraph().valid`.

### Phase 4 — AI engine
**Objective:** optional enhancement, provably safe. **Files:** `src/main/ai/{detect,invoke,passes,contracts}.js`. **Tasks:** `detectAgents()` (which of `claude`/`codex` on PATH + signed in — cheap probe); `invokeAgent(engine,prompt,{timeoutMs,signal})` spawning argv (`claude`,`-p`,prompt,`--output-format`,`json` | `codex`,`exec`,prompt), capturing stdout, timeout+kill; `enhanceGraph()` runs 4 chunked passes, each: build prompt from `contracts.js`, invoke, extract fenced JSON, `ajv` validate, **retry once** on parse/validate failure, else **degrade** to deterministic values + log. **Deliverable test:** with a **mock CLI** (fixture script emitting canned good JSON, chatty-with-JSON, and broken JSON), assert: good→applied; chatty→parsed; broken→retried→degraded; timeout→degraded; **graph never invalid, never throws.**

### Phase 5 — Review workspace UX (the heart)
**Objective:** work the queue to zero on a real document. **Files:** `src/renderer/routes/Review.svelte`, `src/renderer/lib/{workflow,pdfHighlight}.js`, components `QueueList`, `ItemCard`, `PdfPage`, `StatusPicker`, `ContextRail`, `ProgressRing`. **Tasks:** derive queue groups (missing source / needs pincite / unconfirmed cross-ref / reused source / support unverified) from graph+workState; focused item with claim+footnote+source+"what's uncertain"; render the actual PDF page (`pages/<n>.png`) with bbox highlight overlay; status picker with **cascade** (page→claims+footnotes, claim→footnotes, source→footnotes); autosave (debounced) → `saveWorkState`, merge on load, flush on quit; keyboard (`j/k`, number keys). **Deliverable test:** Playwright — import fixture, resolve items, counter decrements to zero, reload persists.

### Phase 6 — Map view
