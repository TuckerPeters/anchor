# Anchor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Anchor — a local, private, double-click desktop workbench that reconstructs the provenance chain (PDF page → claim → footnote → source → cross-reference) for an expert report and guides a non-technical operator through resolving every uncertain citation before handoff.

**Architecture:** One self-contained **Electron** app. Svelte + Vite renderer (workbench UI). Node main process (local JSON store, job runner, IPC, child-process orchestration). **pdf.js runs inside Electron** — a hidden window for extraction (text + positions + page dims) and the Review renderer for live page display — so there is **no native canvas dependency and no pre-rendered PNG artifacts**. A deterministic graph baseline always runs and is the hero path; optional AI passes *augment* it (never overwrite) by driving local, subscription-authenticated CLI agents (`claude -p` / `codex exec`) headlessly, prompt piped via **stdin**, output validated + retried once + degraded to deterministic. JSON file storage, no database.

**Tech Stack:** Electron (latest stable), Svelte 5, Vite (via `electron-vite`), Node 20+, `pdfjs-dist`, `fflate` (unzip DOCX), `fast-xml-parser` (`preserveOrder:true`), `diff-match-patch` (claim→page alignment), `ajv` (schema validation), `electron-builder`, Vitest (unit), Playwright + `_electron` (e2e). Apache-2.0.

## Global Constraints

- **License:** Apache-2.0. Every source file header: `// Copyright © 2026 Tucker Peters. Authored independently, in the author's personal time. Licensed under the Apache License, Version 2.0.` `NOTICE` restates authorship.
- **Privacy is layered and honest:** the app itself makes **zero** network calls and runs no author-operated server. The *one* exception is the optional AI step, which sends document text to Anthropic/OpenAI via the user's CLI agent — this must be disclosed **at the point of enabling AI**, per-document, never silently (see AI egress consent). "100% local" language only appears where it is literally true.
- **AI is optional and always a labeled suggestion.** Every feature works in the deterministic baseline ("Standard review"). AI *augments* the graph via a separate `ai` overlay; deterministic fields are never overwritten. AI passes never crash, hang, or block: they time out, validate, retry once, then degrade and log. The baseline is named positively and never styled as an error/"not connected."
- **Data integrity for privileged legal work:** node ids are **content-derived and stable across re-analysis**; a graph-version + reconciliation pass protects review state on rebuild; status **cascade never silently overwrites** a differing status (esp. `blocked`) and is always single-undo.
- **Security:** child processes spawned with argv arrays + prompt via stdin (never a shell string, never prompt-in-argv); process-group kill on timeout; login-shell PATH resolution for agent binaries; realpath-validated `anchor://` custom protocol for local assets (no `file://`); uploads validated by **magic bytes** + size cap + bounded decompression + sanitized basename; Electron `contextIsolation:true`, `sandbox:true`, `nodeIntegration:false`, `setWindowOpenHandler` deny, navigation locked to app origin, `<webview>` disabled, CSP set.
- **Non-technical UX:** first launch lands on Documents + a demo, never Setup. Setup is always skippable. Every error is plain-English + a recovery action. No terminal after one-time install. No fake spinners.
- **Naming:** product = "Anchor"; storage root = `<app.getPath('userData')>/documents/<id>/`.

## Shared data & IPC contracts (FROZEN — every phase depends on these)

JSDoc typedefs live in `shared/types.js`. **These reflect the eng + design review fixes and must not drift.**

### Node ids — content-derived, stable across re-analysis (`graph/ids.js`)
- page: `p<pageNumber>` — `p1`
- footnote: `f<displayNumber>` where display number = order of first reference occurrence — `f3`
- source: `s-<slug(key)>`; key from register, else `s-<slug(title||firstUrl||sha1(rawText).slice(0,8))>`
- claim: `c-<paraId>` where paraId = `w14:paraId` if present, else `pidx<zeroPaddedIndex>`
- target: `t-<slug(label)>`
- Collisions resolved by appending `-2`, `-3`, … deterministically (document order).
- `slug(s)` = NFKC → lowercase → `[^a-z0-9]+`→`-` → trim, cap 48 chars.

### Graph (`graph.json`, read-only product data)
```jsonc
{
  "documentId":"string", "generatedAt":"ISO-8601", "graphVersion": 1,
  "generator": { "version":"string", "ai": { "engine":"claude|codex", "model":"string" } | null },
  "stats": { "pages":0,"claims":0,"footnotes":0,"sources":0,"targets":0,"edges":0 },
  "inputs": { "docx":"filename|null","pdf":"filename|null","register":"filename|null",
              "textLayer": true, "warnings": [] },   // warnings e.g. "scanned-pdf","endnotes-found","single-file"
  "nodes": {
    "pages":     [{ "id":"p1","number":1,"width":612,"height":792,"text":"normalized page text" }],
    "claims":    [{ "id":"c-1A2B","paraId":"1A2B","pageId":"p3|null","text":"",
                    "footnoteRefs":["f3"],"sourceIds":["s-smith2021"],
                    "anchor": { "state":"located|approximate|none","score":0.0,
                                "bbox": { "x":0,"y":0,"w":0,"h":0 } } }],   // bbox 0–1, TOP-LEFT origin
    "footnotes": [{ "id":"f3","number":3,"role":"support|cross-reference|note","rawText":"",
                    "sourceIds":["s-smith2021"],"candidateTargetIds":["t-iii-b"],"claimIds":["c-1A2B"],
                    "class":"paper|docs|repo|web|case|screenshot|cross-reference|missing|note",
                    "citationNeed":"",
                    "ai": { "class":"", "citationNeed":"", "candidateTargetIds":[], "confidence":0.0, "engine":"claude" } | null }],
    "sources":   [{ "id":"s-smith2021","key":"Smith2021","class":"paper","title":"","url":[],
                    "localCandidate":"","auditNote":"","citationNeed":"",
                    "claimIds":["c-1A2B"],"footnoteIds":["f3"],
                    "ai": { "class":"","citationNeed":"","confidence":0.0,"engine":"claude" } | null }],
    "targets":   [{ "id":"t-iii-b","label":"§ III.B","note":"","footnoteIds":["f3"] }]
  },
  "edges": [{ "id":"e1","type":"page-claim|claim-footnote|claim-source|footnote-source|footnote-target",
              "from":"p3","to":"c-1A2B","meta":{} }]
}
```
**AI overlay rule:** the `ai` object holds *suggested* values only. Deterministic top-level fields are the source of truth and are never overwritten. The UI badges any `ai.*` value as an "AI suggestion" and shows the deterministic value beneath. `ai:null` ⇒ pure baseline.

### WorkState (`work-state.json`, mutable overlay)
```jsonc
{ "documentId":"string", "graphVersion":1, "updatedAt":"ISO-8601",
  "nodes": { "<nodeId>": {
     "status":"todo|in-progress|done|blocked",
     "substep": null,                              // when in-progress: "source-found"|"support-verified"|"draft-written"
     "note":"", "resolution":"",                   // resolution = the structured finding for handoff
     "updatedAt":"ISO-8601", "seq": 0              // monotonic per-node client sequence; merge tiebreak
  } },
  "orphans": [ { "nodeId":"", "prev":{}, "reason":"removed-on-rebuild" } ]  // surfaced, never deleted
}
```
**Cascade safety (behavioral, enforced in `renderer/lib/cascade.js` + verified by test):** cascading a parent status touches a child **only if** the child is `todo`; it never overwrites `in-progress|done|blocked`. Overriding that requires an explicit confirmed "apply to all N (X will change, incl. Y blocked)" action. Every cascade is one atomic undo (UndoToast).
**Reconciliation (`store/reconcile.js`):** on load, if `workState.graphVersion !== graph.graphVersion`, keep statuses whose ids still exist; move vanished-id statuses to `orphans` (never delete) and surface a "N items no longer match the document" banner.

### Document (`document.json`)
```jsonc
{ "id":"string","title":"string","status":"new|extracting|ready|failed",
  "createdAt":"ISO-8601","updatedAt":"ISO-8601","graphVersion":1,
  "inputs": { "docx":null,"pdf":null,"register":null }, "stats": {},
  "ai": { "used": false, "engine": null, "lastUsedAt": null },      // egress marker (design B2)
  "flags": { "textLayer": true, "scanned": false, "singleFile": false },
  "isDemo": false }
```

### Job (`jobs/<jobId>.json`) + checkpoint artifact
```jsonc
{ "id":"string","documentId":"string","type":"build|enhance",
  "status":"queued|running|done|failed",
  "progress": { "phase":"","pct":0,"message":"" },
  "steps": [{ "name":"extract","status":"pending|running|done|failed","detail":"" }],
  "checkpoint": null,                              // mirror of jobs/<id>/checkpoint.json when enhancing

```
`jobs/<id>/checkpoint.json` = `{ "pass":"classify","chunkCursor":90,"total":130,"partialGraph": {…} }`, written after each validated chunk so a rate-limit/quit resumes.

## Phases 2–10 — task lists (step-level TDD authored at phase start)

### Phase 2 — Extractors
**pdfService.js** drives a hidden pdfworker window that loads `pdfjs-dist`, and for each page runs `getTextContent()` → items with `{str,x,y,w,h}` from `transform` + page `viewport` dims, plus a normalized full-page text string; returns `{pages:PageData[], textLayer}`. **Scanned detection:** if total extracted text across pages < threshold, set `textLayer:false` + warning `scanned-pdf`. **docx.js:** `fflate` unzip → `fast-xml-parser` (`preserveOrder:true`, preserve whitespace) over `word/document.xml` + `word/footnotes.xml`; **filter** `w:type="separator"|"continuationSeparator"` (ids -1/0); derive footnote **display number from reference order**, not `w:id`; **recurse** nested run containers (`w:hyperlink`,`w:smartTag`,fields); **exclude** `w:delText` (tracked deletions), include `w:ins`; detect `word/endnotes.xml` → warning `endnotes-found`; prefer `w14:paraId`. **register.js:** Markdown-table → `SourceRaw[]`. **Deliverable test:** unit assertions on `test/fixtures` (the real `Schedule_H_Preexisting_IP.*`, a tracked-changes DOCX, an endnote DOCX, a scanned PDF); scanned + endnote warnings fire; footnote numbers match display order.

### Phase 2b — Claim→page anchoring (TOP RISK, own it here)
**anchor.js** `anchorClaims(graph, pages)`: normalize both sides (NFKC, lowercase, collapse whitespace, strip `U+00AD`, de-hyphenate line-end breaks); run **one global `diff-match-patch` alignment** of the concatenated DOCX claim/body stream against the concatenated per-page PDF text streams; from the alignment derive, per claim, its **pageId** (which page the matched span falls in) and the **union bbox** of matched items, converted to **normalized 0–1 TOP-LEFT** coords (apply the pdf.js y-up→top-left flip once, here). Assign `anchor.state`: `located` (score ≥ hi), `approximate` (mid), `none` (< lo → no bbox, page-only). **Deliverable test:** on the real `Schedule_H` report + an adversarial fixture (hyphenation, running headers, ligatures, intermixed footnote text), ≥ target of claims are `located` with correct pageId and a bbox overlapping the true text; every unmatched claim degrades to `none` (never a wrong bbox). Metrics logged.

### Phase 3 — Deterministic graph builder
**build.js** consumes extractor output: assign **content-derived ids** (`ids.js`); match footnote refs → claims; classify footnote `class` + `citationNeed` heuristics (`classify.js`); link sources (register + footnote text); build all edge types; compute stats; then `anchorClaims`; `validateGraph`. `ai` overlays are `null` here. **Deliverable test:** on fixtures, exact stats, every edge endpoint resolves, ids stable across two runs, `validateGraph().valid`.

### Phase 4 — AI engine
**detect.js** resolves `claude`/`codex` via **login-shell PATH** (`$SHELL -lic 'command -v …'`), handles Windows `.cmd` shims; `signedIn` = `codex login status` / claude credential-presence heuristic (never spends quota). **invoke.js** `invokeAgent` spawns detached (`claude -p --output-format json --allowedTools "" --max-turns 1` / `codex exec --json -`), **prompt via stdin**, `timeoutMs` → **process-group kill**, AbortSignal wired. **jsonExtract.js** envelope-aware: claude `{type:result,result:"```json…"}` two-level; codex terminal `agent_message`; fallbacks whole-parse → last ```json fence → balanced-brace. **passes.js** `enhanceGraph` runs chunked passes (classification, citation-need, cross-ref, claim-detection) writing **`ai.*` overlays only**; each chunk: prompt → invoke → extract → ajv-validate → retry once → else degrade+log; write `checkpoint.json` after each validated chunk; resume from cursor. **Deliverable test (mock CLI):** fixture script emitting the **real claude envelope** (good, chatty-with-fence, broken), >32KB stdin payload, timeout, is_error; assert applied/parsed/retried/degraded, checkpoint resume from mid-run, graph never invalid, never throws, deterministic fields untouched.

### Phase 5 — Review workspace UX (the heart)
Queue groups (missing source · needs pincite · unconfirmed cross-ref · reused source · support unverified) from graph+workState. **Task-first** focused item: (1) plain-English task + how-to-resolve, (2) **live pdf.js page** (loaded from `anchor://doc/<id>/pdf`) with **confidence-aware highlight** (`located` solid / `approximate` soft "near here" / `none` page-only "couldn't pinpoint" banner), (3) supporting detail (footnote raw text, source, linked claims); **AI values badged** via `AiBadge` with deterministic value revealable. 4-status `StatusPicker` (+ in-progress substeps) with **safe cascade** (`cascade.js`: todo-only, confirm to override, single UndoToast) and a `resolution` field. Autosave (debounced, bumps `seq`) → `saveWorkState`; reconcile + orphan banner on load. Context rail default-open on wide windows. Keyboard = pure accelerator (visible controls always; `?` cheat-sheet; destructive keys guarded). "Standard review" positive baseline; no red "not connected." **Deliverable test:** Playwright — open demo, resolve items, counter → 0, blocked-preserving cascade, undo restores, reload persists, low-confidence claim shows page-only banner not a wrong highlight.
